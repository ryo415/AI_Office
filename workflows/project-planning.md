# Project Planning Workflow

## Purpose

Turn an idea or goal into an actionable project.

## When to Use

- The user wants to start something
- The work requires multiple steps
- The work should be tracked over time
- The idea may become a Notion project, GitHub issue, or project document

## Roles

- Chief of Staff
- PM
- Reviewer
- Secretary, if structured output is needed

## Steps

1. Define the project goal.
2. Clarify the background and motivation.
3. Define success criteria.
4. Define scope and out-of-scope items.
5. Break the project into milestones.
6. Break milestones into tasks.
7. Identify risks, blockers, and dependencies.
8. Decide the next action.
9. Prepare a project document using `templates/project.md`.
10. Add or update the project index in `memory/active-projects.md` if the project is active.

## Output

Use `templates/project.md`.

## Rules

- Keep the first version small.
- Prefer a minimum viable project.
- Avoid overengineering.
- Every project must have a clear next action.
- Keep detailed project state in `projects/`, not in `memory/`.
