# Research Workflow

## Purpose

Research a topic and produce a decision-ready summary.

## When to Use

- The user asks for comparison
- The user asks about recent tools, services, specs, laws, prices, or trends
- The user needs evidence before deciding
- The topic is unfamiliar or fast-changing

## Roles

- Researcher
- Reviewer
- Chief of Staff

## Steps

1. Define the research question.
2. Define evaluation criteria.
3. Gather information.
4. Compare options.
5. Identify risks and uncertainty.
6. Recommend a practical option.
7. Prepare a research report using `templates/research-report.md`.

## Output

Use `templates/research-report.md`.

## Rules

- Prefer official sources.
- Cite sources when available.
- Separate facts from assumptions.
- State uncertainty clearly.
- Use Markdown source links that future tools can parse.
