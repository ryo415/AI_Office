# Output Workflow

## Purpose

Convert work into a specific reusable format.

## When to Use

- The user wants a Notion-ready document
- The user wants a GitHub issue draft
- The user wants Markdown
- The user wants a task list
- The user wants a project document
- The user wants a weekly review

## Roles

- Secretary
- Chief of Staff
- Reviewer, if important

## Steps

1. Identify the output destination.
2. Select the correct template.
3. Convert the content.
4. Check formatting.
5. Ask for confirmation before external write actions.

## Supported Outputs

- Markdown
- Notion-ready Markdown
- GitHub issue draft
- Task list
- Research report
- Project document
- Decision log
- Weekly review
- Review report
- Inbox item

## Rules

- Do not write to external tools without confirmation.
- Prefer copy-pasteable output.
- Keep headings consistent.
- Prefer templates in `templates/` over ad hoc formatting.
