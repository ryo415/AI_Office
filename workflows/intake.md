# Intake Workflow

## Purpose

Convert a vague user request into a clear working request.

## When to Use

- The user has a rough idea
- The goal is unclear
- The output format is unclear
- The request may become a project
- Multiple interpretations are possible

## Steps

1. Restate the user's request.
2. Identify the likely goal.
3. Identify missing information.
4. Decide whether to ask questions or proceed with assumptions.
5. Select the next workflow.

## Rules

- Do not ask too many questions.
- If the task is still actionable, proceed with assumptions.
- Ask questions only when the answer would significantly change the result.
- Save rough, unresolved material in `inbox/` instead of forcing it into a project.

## Output

```markdown
## Understanding

## Likely Goal

## Assumptions

## Missing Information

## Recommended Next Workflow

## Next Action
```
