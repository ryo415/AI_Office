# Preferences

## General

- Prefer practical and concrete answers.
- Prefer starting with small working systems.
- Avoid unnecessary abstraction.
- Keep outputs easy to edit with Codex.

## Technical

- Prefer Markdown for documentation.
- Prefer Git-managed files.
- Prefer CLI-friendly workflows.
- Prefer tools that work well on Arch Linux.
- Prefer reversible and inspectable changes.

## AI_Office

- Use Chief of Staff as the default interface.
- Use workflows for repeatability.
- Use templates for structured outputs.
- Do not overuse roles.
- Ask before external write actions.
- Keep Markdown as the source of truth.
- Add integration-specific fields only when they are useful without the integration.
