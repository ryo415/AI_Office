# Task Breakdown Skill

## Purpose

Break a goal into actionable tasks.

## Input

- Goal
- Background
- Constraints
- Deadline, if any
- Desired output

## Rules

- Tasks should start with verbs.
- Tasks should be small enough to complete.
- Separate "decide", "research", "implement", and "review" tasks.
- Always include one next action.
- Keep task ownership and priority optional unless needed.

## Output

```markdown
## Goal

## Milestones

## Tasks

## Next Action

## Blockers

## Notes
```
