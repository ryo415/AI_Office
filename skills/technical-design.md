# Technical Design Skill

## Purpose

Create a practical technical design for a feature, tool, automation, or system.

## Input

- Goal
- Users
- Environment
- Constraints
- Existing system
- Required integrations

## Rules

- Prefer simple architecture.
- Avoid premature abstraction.
- Include verification.
- Include operational concerns when relevant.
- Keep external integration assumptions explicit.

## Output

```markdown
## Goal

## Requirements

## Non-Goals

## Architecture

## Components

## Data Flow

## Files or Modules

## Risks

## Verification

## Next Actions
```
